# 上传文档前端问答评测报告

日期：2026-07-11

## 评测范围

- 通过项目真实前端代理 `http://127.0.0.1:8081` 调用登录与问答接口。
- 共 40 个问题，覆盖电子劳动合同、外服云微助手、工单系统、派单截止表、分公司进度表和入职信息表。
- 问题、人工参考答案与自动断言见 `frontend_live_eval_cases.json`。
- 每个用例创建独立临时会话，并在完成后删除；没有出现会话清理失败。
- 两张重复的 AI 对话截图不作为业务知识；入职表评测只检查字段名，不使用个人信息。

## 结果

初次全量实测为 23/40，耗时 819.9 秒。主要缺陷是表格复合字段遗漏、Wiki 后页事实被上下文预算截断、字段问题混入无关表格，以及 UI 引用候选未过滤。

修复后全量运行的自动字面规则结果为 36/40，耗时 778.8 秒。4 个失败答案经人工核验均语义正确，失败原因是组合词不同，例如“个人身份认证”“批量数据导入导出”“自身所属职能模块”。补充等价表达后，4 个用例均已定向复测通过。

## 已修复问题

1. 表格复合问题现在能同时投影社保、医保、公积金截止日及操作规则、备注和证明材料。
2. “银行和社保公积金账户”会展开为银行账户与社保公积金账户两个字段。
3. “分别怎样/分别如何”等自然问法会进入结构化表格路由。
4. Wiki 上下文优先放入相关来源，读取关联原始 chunk，避免 360 字 quote 截断后页事实。
5. 字段问题命中 form 文档后不再用分公司、派单表补足跨文档多样性；没有 form 时仍保留 table 兜底。
6. 入职字段题使用确定性抽取，避免 LLM 随机漏掉字段。
7. 普通问答引用只来自实际答案上下文，最多 8 条、同一文档最多 4 条，并按片段或表格行去重。
8. 工单答案不再引用无关的 `image` 截图，分公司和入职题不再出现引用泛滥。

## 剩余风险

回答正文由 LLM 生成时会出现正常的同义改写。当前自动评测采用人工维护的等价短语组，不是语义裁判；新增问题时仍应保留人工参考答案，并对自动失败做一次证据核验。
