# 业务规则回归清单

> 用途：每次修改后，根据改动范围选择性执行。涉及聊天、知识库、PageIndex、后台页面时必须至少跑对应小项。

## 基础服务检查

### 后端

- 后端端口：`8000`
- 健康检查：
  - `http://127.0.0.1:8000/api/health`
- 后端编译：
  - 工作目录：`backend`
  - 命令：`python -m compileall app`

### 前端

- 前端端口：`5174`
- 禁止操作：`5173`
- 前端页面检查：
  - `http://127.0.0.1:5174/chat`
  - `http://127.0.0.1:5174/admin`
- 前端构建：
  - 工作目录：`frontend`
  - 命令：`npm run build`

## 聊天问答回归

### 普通对话不应调用知识库

输入：

```text
你好
```

预期：

- `mode=chat`
- 不显示知识库来源
- 不显示“未在知识库中找到依据”

输入：

```text
你是谁？你可以做什么
```

预期：

- 直接回答助手身份/能力
- 不调用知识库
- 不显示来源片段

### 无上下文编辑/整理类请求

输入：

```text
把内容整理成表格
```

预期：

- `mode=chat`
- `retrieval_backend=none`
- AI 应回复类似“请把需要整理的具体内容发给我”
- 不检索知识库
- 不引用无关文档
- 不报 `Failed to fetch`
- 不出现空白 AI 卡片

输入：

```text
为什么会这样吗，帮我整理成表格
```

预期：

- 如果无上下文，应追问需要整理的内容或说明需要具体材料
- 不应因为“表格”二字检索无关知识库



### 北仑分公司开设进度统计口径

输入：

```text
目前北仑在哪些城市开了分公司，以有分公司名称的为准。
```

预期：

- 走表格检索。
- 以 `当前进度-4.开设公司名称` 有效为准：非空且不是 `未开设/无/暂无/否` 即计入。
- 不要求名称必须包含“分公司”。
- `宁波北仑 -> 外服（浙江）企业服务有限公司` 计入。
- 不自动叠加银行账户、社保公积金账户完成过滤。
- 真实库最终统计口径为 **36**。

### 明确知识库/文档请求

输入：

```text
总结我现在可读的文档
```

预期：

- 走知识库/可读文档总览
- 返回当前用户可读文档清单摘要
- 正文不出现 `[来源1]`、`[来源2]` 这类标记
- 来源在独立来源面板展示

输入：

```text
把文档内容整理成表格
```

预期：

- 允许检索知识库
- 有来源面板
- 不引用明显无关材料

### 来源展示

预期：

- 正文不显示突兀 `[来源1]`
- 来源按钮文案：
  - `查看来源`
  - `查看文档清单`
  - `相关来源`
- “打开文档/下载文档”应带鉴权请求，不直接裸开 `/api/...`
- Office 文件通常显示下载行为，PDF/图片/TXT/CSV 可尝试浏览器打开

## 知识库 / PageIndex 回归

### 上传限制

- 默认上传限制：`200MB`
- `.env.example` 和后端默认配置应一致

### PageIndex 路径

- 支持类型：
  - PDF
  - Markdown
  - Word docx
  - Excel xlsx
  - PPT pptx
  - CSV
  - TXT
  - 图片
- PDF/Markdown 可走官方 PageIndex 原生模式
- 其他类型可先抽取文本/OCR 转 Markdown 再构建 PageIndex
- 扫描 PDF 应走 PDF OCR / 视觉解析

### PageIndex 状态

后台管理页面应能看到：

- 是否已构建高级索引
- 结构树
- 重建高级索引入口

编辑片段后应提示：

```text
请重建高级索引
```

### PageIndex 兼容性

- `chunk_index` 可能是字符串：`pageindex:0000`
- 结构化摘要、排序、来源展示都不能强制 `int(chunk_index)`

## 后台管理页面回归

### 页面滚动

后台管理页数据变多时应可向下滚动。

预期：

- `admin` 页面不能被聊天页全局 `overflow:hidden` 锁死
- 表格、任务、结构树区域可访问

### 上传/解析状态

上传文件后应显示：

- 上传成功
- 后台任务状态
- 解析中/完成/失败
- 分片数量
- PageIndex 状态

## 前端稳定性回归

### 5174 服务

如果出现：

```text
Failed to fetch
```

优先检查：

- `5174` 是否还在 Listen
- `8000` 是否健康
- Vite 代理 `/api/health` 是否可用

### 空白 AI 卡片

不允许出现：

```text
AI 助手
复制
有帮助
不够好
```

但正文空白。

如果后端异常，前端至少要显示错误原因或可读提示。

## 推荐最小测试组合

修改聊天/检索/模型路由后，至少测试：

```text
你好
你是谁？你可以做什么
把内容整理成表格
总结我现在可读的文档
把文档内容整理成表格
```

修改后台/知识库/PageIndex 后，至少测试：

```text
后台页面可滚动
上传小 TXT/CSV
查看分片
查看 PageIndex 结构树
重建高级索引
总结我现在可读的文档
```
