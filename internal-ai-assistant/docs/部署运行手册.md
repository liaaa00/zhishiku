# 部署运行手册

本文记录当前项目的实际部署方式、配置项、启动、验证、备份和回滚步骤。

## 1. 部署拓扑

```text
用户浏览器
  ↓
frontend: Nginx + Vue SPA，默认宿主机端口 8080
  ↓ /api/* 反向代理
backend: FastAPI，容器内 8000
  ↓
SQLite: /app/data/app.db
uploads: /app/data/uploads
PageIndex: /app/data/pageindex
Qdrant: http://qdrant:6333，volume qdrant_storage
  ↓
OpenAI-compatible LLM / Embedding 服务
```

Compose 服务：

- `frontend`：构建 `./frontend`，暴露 `${FRONTEND_PORT:-8080}:80`。
- `backend`：构建 `./backend`，读取 `.env`，挂载 `./backend/data:/app/data`。
- `qdrant`：镜像 `qdrant/qdrant:v1.15.4`，持久化 volume `qdrant_storage`。

## 2. 初次部署

### 2.1 准备 `.env`

从 `.env.example` 复制：

```powershell
Copy-Item .env.example .env
```

生产建议至少设置：

```env
APP_ENV=production
DEFAULT_ADMIN_USERNAME=admin
DEFAULT_ADMIN_PASSWORD=<强口令>
JWT_SECRET=<高强度随机字符串>
OPENAI_API_KEY=<LLM Key>
OPENAI_BASE_URL=https://api.deepseek.com
CHAT_MODEL=<实际模型名>
VECTOR_BACKEND=qdrant
QDRANT_URL=http://qdrant:6333
QDRANT_COLLECTION=internal_ai_chunks
EMBEDDING_PROVIDER=openai-compatible
EMBEDDING_BASE_URL=<embedding endpoint>
EMBEDDING_MODEL=<embedding 模型>
EMBEDDING_API_KEY=<embedding key>
CORS_ORIGINS=https://<你的域名>
MAX_UPLOAD_MB=200
```

注意：

- `.env` 不能提交 Git，也不要发到普通聊天或文档。
- `APP_ENV=production` 会拒绝默认 JWT、默认管理员密码和 local-hash embedding。
- Docker Compose 内后端访问 Qdrant 应使用 `http://qdrant:6333`。

### 2.2 只读预检

```powershell
python -X utf8 scripts\preflight_check.py
```

预检只读取文件和路径，不会启动服务、修改数据库或上传资料。若出现 `[FAIL]`，先修复再启动。

### 2.3 启动

```powershell
docker compose up -d --build
```

如果 8080 被占用：

```powershell
$env:FRONTEND_PORT='8081'
docker compose up -d --build
```

### 2.4 查看状态

```powershell
docker compose ps
docker compose logs --tail=200 backend
docker compose logs --tail=200 frontend
docker compose logs --tail=200 qdrant
```

健康检查：

```powershell
Invoke-WebRequest http://localhost:8080/api/health
```

浏览器访问：

- 聊天端：`http://localhost:8080/chat`
- 管理端：`http://localhost:8080/admin`

## 3. 日常运行

### 3.1 管理员首登

1. 用 `.env` 中的 `DEFAULT_ADMIN_USERNAME` / `DEFAULT_ADMIN_PASSWORD` 登录。
2. 立即在管理后台创建或审批实际管理员/普通用户。
3. 建立用户组。
4. 后续文档按用户组授权。

### 3.2 上传知识库资料

1. 进入 `/admin`。
2. 上传 PDF、docx、pptx、xlsx、csv、txt、markdown 或图片。
3. 选择知识范围：正式资料用 `production`，试验资料用 `test`。
4. 检查文档分类：表格、员工指南、工单、表单、政策、通用等。
5. 等后台状态变为 `ready/searchable`。
6. 对表格类文档执行搜索测试和业务口径确认。
7. 设置文档可见用户组。

### 3.3 重建索引

适用场景：

- 换了 embedding 模型或维度。
- 文档权限变更后需要同步向量 payload。
- 手动编辑 chunk 后需要同步索引。
- Qdrant 数据丢失或迁移。

后台可使用向量重建入口，也可调用：

```text
POST /api/admin/vector/reindex
```

重建后必须运行搜索测试和真实评测。

## 4. 备份

### 4.1 需要备份的对象

- `backend/data/app.db`
- `backend/data/uploads/`
- `backend/data/pageindex/`
- Docker volume `qdrant_storage`
- `.env`（进密钥管理系统，不进 Git）

### 4.2 建议备份窗口

在低峰期执行：

1. 暂停批量上传和向量重建。
2. 备份 SQLite 与 `backend/data`。
3. 备份 Qdrant volume。
4. 记录当前 Git commit、镜像版本、`.env` 版本号。
5. 恢复到临时环境验证。

可先使用项目内只读源数据备份脚本生成 `backend/data` 归档：

```powershell
.\scripts\backup-data.ps1
# 如确需把 .env 放入离线加密备份，可显式加参数：
# .\scripts\backup-data.ps1 -IncludeEnv
```

注意：该脚本默认不包含 `.env`，也不包含 Docker named volume `qdrant_storage`；Qdrant 仍需按服务器实际部署方式单独备份。

### 4.3 恢复验证

恢复完成后至少验证：

```powershell
Invoke-WebRequest http://localhost:8080/api/health
```

并在页面验证：

- 管理员能登录。
- 后台文档列表存在。
- 文档来源可以打开。
- Qdrant 状态正常。
- 至少 3 个高频问题能命中正确来源。

## 5. 升级与回滚

### 5.1 升级前

```powershell
git status --short
cd backend
python -X utf8 -m pytest tests/unit -q
python -X utf8 tests/qa_retrieval_eval_runner.py --real-db --cases tests/retrieval_eval_real_cases.json
cd ..\frontend
npm run build
```

通过后再构建镜像。

### 5.2 升级

```powershell
docker compose up -d --build
```

升级后检查：

- `/api/health` 返回 `{"ok": true}`。
- `/chat`、`/admin` 可打开。
- 管理后台文档状态正常。
- 关键问题可回答。

### 5.3 回滚

1. 切回上一 Git commit 或上一镜像。
2. 恢复升级前的 `backend/data` 和 Qdrant 备份。
3. 启动服务。
4. 执行恢复验证。

注意：如果新版本已经修改了 SQLite 结构，优先恢复升级前备份，而不是直接用旧代码读取新库。

## 6. 监控与排障

### 6.1 常用日志

```powershell
docker compose logs --tail=200 backend
docker compose logs --tail=200 frontend
docker compose logs --tail=200 qdrant
```

重点关注：

- 后端启动时是否有“生产环境安全检查未通过”。
- 文档解析任务是否 failed。
- Qdrant 是否连接失败。
- LLM/embedding 是否超时或鉴权失败。
- 登录是否频繁 429。

### 6.2 常见问题

| 现象 | 可能原因 | 处理 |
|---|---|---|
| 生产启动失败 | 默认 JWT/管理员密码/local-hash embedding | 修改 `.env` 后重启 |
| 上传成功但不可检索 | 后台任务失败、OCR 不足、embedding 失败 | 看任务状态和 backend 日志，必要时重新解析 |
| 表格统计不准 | 列语义别名或业务口径未确认 | 进入表格 schema/别名确认，并补评测用例 |
| 问答没有来源 | 检索未命中或权限过滤 | 用管理员搜索测试，确认文档权限和评测 |
| 普通用户看不到资料 | 文档未分配该用户组 | 后台调整文档权限后重建向量 |
| Qdrant 丢数据 | volume 未持久化或迁移遗漏 | 从 SQLite 分块重建向量，并恢复备份 |

## 7. 当前已知限制

- 默认 SQLite 适合内部小规模试运行；并发和审计要求更高时应规划 PostgreSQL。
- 登录限流当前为单进程内存实现；多实例部署时应替换为 Redis 限流。
- PageIndex 官方实现是可选增强，未配置时会回退轻量结构树。
- 扫描件/OCR 质量取决于文档清晰度和视觉模型配置。
- 评测集不能替代业务验收；新增业务口径必须写入评测和文档。
