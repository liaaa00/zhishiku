## 【进行中 / 待解决的 Bug】🔧

> 这些是重构完成后还没细测的边缘情况，或在代码审计中发现的隐患。

### 安全
- [ ] **[高] 密码使用 SHA256 而非 bcrypt**：`passlib[bcrypt]` 已安装但 `security.py` 实际调用 `hashlib.sha256`。SHA256 无盐、计算过快，彩虹表攻击风险。用户密码安全等级不足。
- [ ] **[中] CORS 全开**：`allow_origins=["*"]`。生产环境应限制为实际域名。
- [ ] **[中] 无 Token 刷新机制**：24 小时过期后需重新登录，用户体验差。

### 前端
- [x] **[中] 无路由守卫**：未登录访问 `/chat` 或 `/admin` 会跳转 `/login`，非管理员访问 `/admin` 会跳转 `/chat`。
- [x] **[中] 无全局 401 拦截**：Token 过期后 API 返回 401，前端会清理 token/user 并自动跳转登录页。
- [ ] **[低] 聊天页输入框长文本无自动扩展**：textarea 未设 `autosize`，长问题需要手动滚动。

### 后端
- [ ] **[中] main.py 巨石文件**：1576 行单文件包含所有 API 路由、业务逻辑、HTML 模板和 JavaScript。难以维护和测试。
- [ ] **[中] 无数据库迁移**：表结构变更依赖 `Base.metadata.create_all`，无 Alembic 版本控制。
- [ ] **[低] Embedding 默认 local-hash**：基于 MD5 的稀疏向量，检索精度有限。生产应切换到 OpenAI-compatible embedding。
- [ ] **[低] Qdrant 回退为静默**：Qdrant 不可用时自动回退 SQLite 向量检索，但无告警通知。

### 未完成的重构
- [ ] **[低] Pinia Store 空置**：Pinia 已挂载但代码中无 Store 使用，所有状态都在组件内部管理。
- [ ] **[低] 无单元测试**：仅有 QA 回归测试脚本，无 `pytest` 或 `vitest` 单元测试。
- [ ] **[低] SSR HTML 与 Vue SPA 功能重复**：两个版本的聊天页和管理页功能不完全一致。
