## 【已完成】✅

### 核心功能
- [x] 员工登录/登出（JWT Token 24h 过期，支持临近过期自动刷新）
- [x] 基于知识库的 AI 问答（DeepSeek 模型，RAG 检索增强生成）
- [x] 向量检索（Qdrant 为主，SQLite 向量回退）
- [x] 多格式文档上传与解析（PDF, DOCX, XLSX, CSV, TXT, MD）
- [x] 图片 OCR（通过 DeepSeek Vision API）
- [x] 聊天附件上传（临时文档，仅上传者可见）
- [x] 回答引用来源展示（来源卡片、关键词高亮、定位预览抽屉）
- [x] 用户反馈系统（有帮助/不够好 + 详细分类 + 管理员审核流程）
- [x] 内容合规检测（回答偏离知识库时的警告提示）
- [x] 预设快捷提问卡片
- [x] 拖拽上传文件（聊天页面）

### 人员权限管理（已重构）
- [x] 纯角色管理模型：UserID → GroupID 映射，不冗余存储飞书元数据
- [x] 管理员可创建/编辑用户、分配岗位组
- [x] 管理员可创建岗位组
- [x] 用户登录、密码重置（bcrypt 哈希，兼容旧 SHA256 登录时自动迁移）
- [x] 账号启用/停用、删除用户
- [x] 自我删除/自我停用保护
- [x] 审计日志（用户操作全量记录）
- [x] 已移除自建本地组织架构同步逻辑
- [x] 飞书 JS-SDK 按需加载并接入管理员员工页选人/选部门框架（不持久化飞书元数据，仅保存 username 与 group_ids）

### 文档权限管理
- [x] 文档上传后可分配岗位组（基于组的访问控制）
- [x] 权限变更自动同步 Qdrant payload
- [x] 个人聊天附件仅上传者可见
- [x] 管理员可查看所有托管文档
- [x] 文档重新解析（入后台队列）
- [x] 文档删除（清理文件 + 数据库行 + 向量索引）

### 管理后台
- [x] 5 个 Tab：岗位组 / 员工 / 文档与权限 / 用户反馈 / 模型配置与审计
- [x] 反馈统计概览（总数/待处理/已查看/已解决）
- [x] 反馈详情面板（提问/回答快照 + 来源 + 审核表单）
- [x] 模型配置（API Key, Base URL, Model）
- [x] 向量库状态查看 + 全量重建索引
- [x] Qdrant 健康检查可显示 ready/degraded/local，管理员后台可见 SQLite 回退告警
- [x] 后台任务监控与重试
- [x] 审计日志查看

### 基础设施
- [x] Docker Compose 一键部署
- [x] Nginx 反向代理 + 静态资源强缓存
- [x] Vite 手动分包（vendor-vue, element-plus-core/data/overlay, vendor-axios）
- [x] 环境变量驱动配置（.env）
- [x] 双前端形态（Vue SPA + SSR HTML 回退）
- [x] Qdrant Collection 自动创建
- [x] CORS 改为 `CORS_ORIGINS` 白名单配置，拒绝非白名单 Origin
- [x] FastAPI 路由已从 `main.py` 拆分到 `backend/app/routers/` 按领域维护
- [x] 核心身份/权限接口已补充 Pydantic `response_model`（auth/users/groups），OpenAPI 响应契约可回归验证
- [x] Alembic 初始迁移已接入，可从空 SQLite 数据库执行 `alembic upgrade head`
- [x] 生产环境 embedding 护栏已接入：`APP_ENV=production` 拒绝 local-hash，要求 OpenAI-compatible embedding，并补充检索评测切换流程。
- [x] Pinia 已从空挂载变为实际承载认证状态：`auth` store 统一 token/user/isAdmin/logout，router/login/chat 复用同一登录态。

## 【进行中 / 待解决的 Bug】🔧

> 这些是重构完成后还没细测的边缘情况，或在代码审计中发现的隐患。

### 安全
- [x] **[中] Token 自动刷新机制**：提供 `/api/auth/refresh`，前端在 token 临近过期或 401 重试时自动续签。

### 前端
- [x] **[中] 无路由守卫**：未登录访问 `/chat` 或 `/admin` 会跳转 `/login`，非管理员访问 `/admin` 会跳转 `/chat`。
- [x] **[中] 无全局 401 拦截**：Token 过期后 API 返回 401，前端会清理 token/user 并自动跳转登录页。
- [ ] **[低] 聊天页输入框长文本无自动扩展**：textarea 未设 `autosize`，长问题需要手动滚动。

### 后端
- [x] **[低] Embedding 默认 local-hash**：生产环境已拒绝 local-hash，需配置 OpenAI-compatible embedding；本地仍可用 local 便捷试跑。

### 未完成的重构
- [x] **[低] Pinia Store 空置**：已落地 `frontend/src/stores/auth.ts` 管理登录态；大型页面内部业务状态暂不拆分，避免无收益大改。
- [ ] **[低] 无单元测试**：仅有 QA 回归测试脚本，无 `pytest` 或 `vitest` 单元测试。
- [ ] **[低] SSR HTML 与 Vue SPA 功能重复**：两个版本的聊天页和管理页功能不完全一致。
