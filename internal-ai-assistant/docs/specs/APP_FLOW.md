# APP_FLOW.md — 用户流程文档

> 最后更新：2026-05-29  
> 本文档用大白话记录当前已实现的所有页面、路由和用户导航流程。  
> **不含已废弃的"同步组织架构"流程。**

---

## 1. 路由表

| 路径 | 页面名称 | 渲染方式 | 需要登录 | 需要管理员 |
|------|----------|----------|----------|-----------|
| `/` | 根路径 | 302 重定向到 `/chat` | 否 | 否 |
| `/login` | 员工登录页 | Vue SPA (Element Plus) | 否 | 否 |
| `/chat` | 聊天问答页 | Vue SPA (完整功能，后端可回退 SSR) | 是 | 否 |
| `/admin` | 后台管理页 | Vue SPA (完整功能，后端可回退 SSR) | 是 | 是 |

**注意**：后端 FastAPI 在 `FRONTEND_DIST_DIR` 存在构建产物时，会优先为 `/login`、`/chat`、`/admin` 返回 Vue SPA 的 `index.html`，并通过 `/assets/...` 提供前端资源。仅当未构建或未挂载前端 dist 时，`/chat` 和 `/admin` 才回退到内联 SSR HTML（`CHAT_HTML` 和 `ADMIN_HTML`）。

## 2. 完整用户旅程

### 2.1 未登录用户

```
打开网站 → 自动跳转到 /login
                │
                ├─ 输入账号密码 → 点击"登录"
                │       │
                │       ├─ 成功 → 存 token 到 localStorage → 跳转 /chat
                │       └─ 失败 → 页面显示错误提示（用户名或密码错误 / 账号已停用）
                │
                └─ 提示文字："请使用管理员分配的公司账号登录。"
```

**关键实现细节**：
- 登录表单使用 Element Plus 的 `el-card` + `el-form` + `el-input`
- 密码框支持 `show-password` 切换明文
- 登录成功后将 JWT token 写入 `localStorage('token')`
- 当前**没有注册页面**，新用户只能由管理员在后台创建

### 2.2 已登录普通用户 — 聊天页 (`/chat`)

```
进入 /chat
  │
  ├─ 页面顶部 Header
  │   ├─ "Internal Knowledge Assistant"
  │   ├─ "公司内部 AI 助手"
  │   ├─ 副标题："每次回答都会基于你有权限访问的知识库内容..."
  │   └─ 两个按钮：[后台管理]（仅管理员可见实际跳转） [退出]
  │
  ├─ 首次访问（无消息历史）
  │   ├─ 显示欢迎界面
  │   │   ├─ 大图标 "AI"
  │   │   ├─ "今天想了解什么？"
  │   │   └─ 快捷提问卡片（3 个预设问题按钮）
  │   │
  │   └─ 底部输入区（始终可见）
  │       ├─ textarea 文本框（Enter 发送，Shift+Enter 换行）
  │       ├─ [📎 上传文件] 按钮（支持 PDF/Word/Excel/CSV/TXT/MD/图片）
  │       └─ [发送] 按钮
  │
  ├─ 已有对话
  │   ├─ 消息列表（按时间正序）
  │   │   ├─ 用户消息（右侧对齐，蓝色背景 "我"）
  │   │   └─ AI 消息（左侧对齐，白色背景 "AI 助手"）
  │   │       ├─ 消息正文（Markdown 渲染 / 纯文本）
  │   │       ├─ ⚠️ 内容合规提示（如"警告：回答可能偏离知识库"）
  │   │       ├─ 📄 引用来源区域
  │   │       │   ├─ "引用的知识库文件 · N 条"
  │   │       │   ├─ 每条来源卡片：
  │   │       │   │   ├─ 序号
  │   │       │   │   ├─ 文档标题
  │   │       │   │   ├─ 类型 + 位置（如 "pdf · 第 3 页"）
  │   │       │   │   ├─ 相关度标签（el-tag）
  │   │       │   │   ├─ 匹配原因文字
  │   │       │   │   ├─ 命中片段预览（高亮关键词）
  │   │       │   │   └─ [定位预览] 按钮
  │   │       │   └─ 无命中时显示 el-empty "本次回答未命中可引用文件。"
  │   │       │
  │   │       └─ 👍👎 反馈区
  │   │           ├─ "这条回答需要反馈吗？"
  │   │           ├─ [有帮助] [不够好] 按钮
  │   │           └─ 已提交后显示 "已提交给管理员"
  │   │
  │   └─ 错误提示（el-alert，可关闭）
  │
  └─ 附件上传流程
      ├─ 点击 📎 → 文件选择对话框
      ├─ 支持拖拽上传（拖到输入区）
      ├─ 文件限制 30MB
      ├─ 上传后显示处理状态 chip（可检索 / 已记录 / 失败）
      └─ 附件自动参与后续问题的检索
```

**关键交互**：
- 点击来源卡片的 [定位预览] → 打开 `SourcePreviewDrawer`（右侧抽屉）
  - 显示文档详情、命中片段高亮、[打开知识库文件] / [重新加载预览] 按钮
- 点击 [有帮助] / [不够好] → 弹出反馈对话框，选择原因分类，输入详细说明 → 提交
- 点击 [退出] → 清除 token，跳转到 `/login`

- 反馈审核：点击 [查看详情] → 展开内联详情面板 → 填写备注 + 选择状态 → [保存审核]

### 2.4 页面间导航跳转

```
/login ──(登录成功)──→ /chat
/chat ──([后台管理])──→ /admin
/chat ──([退出])──────→ /login
/admin ──(切换标签)──→ 5 个 Tab 页签（前端无路由切换，同一组件内 el-tabs 切换）
/admin ──(浏览器手动回退)──→ 前一个历史记录
未登录手动访问 /chat 或 /admin ──(路由守卫)──→ /login
普通用户手动访问 /admin ──(路由守卫)──→ /chat
Token 临近过期 ──(Axios 请求拦截)──→ 调用 `/api/auth/refresh` 自动续签
API 401 ──(Axios 响应拦截)──→ 尝试刷新并重试一次；仍失败则清理 token/user 并跳转 /login
```

**注意**：
- 前端路由守卫基于 localStorage 中的 token 与 user.is_admin 做页面级保护；后端 API 仍通过 `require_user` / `require_admin` 做最终权限校验。
- 聊天页**无侧边栏**、**无会话历史列表**，会话通过 URL 参数或默认新建。

## 4. 两种前端形态说明

当前系统存在**双前端**形态：

| 形态 | 触发方式 | 技术栈 | 说明 |
|------|----------|--------|------|
| Vue SPA | Nginx 代理访问，或后端 8000 端口存在 `FRONTEND_DIST_DIR/index.html` 时访问 `/login`、`/chat`、`/admin` | Vue 3 + Element Plus + Vite | 完整交互（反馈对话框、来源预览抽屉等） |
| SSR HTML | 后端 8000 端口缺少前端 dist 时访问 `/chat` 或 `/admin` | FastAPI `HTMLResponse` + 内联 JS | 基础功能回退（发送消息、查看来源、管理文档权限） |

两个形态共享同一套后端 API。Vue SPA 为**主要支持形态**，SSR HTML 为**容灾回退**；生产优先维护 Vue SPA，避免双实现长期分叉。
